## V1.0.9 (2026-03-06) - 删除功能 Bug 修复

### 修复内容

#### 1. 删除确认对话框报错 🔧
- **问题**：点击删除按钮时报错 `Cannot read properties of null (reading 'addEventListener')`
- **原因**：`ui.js:449` 使用 `modal.querySelector('dialog')` 但 `modal` 本身就是 dialog 元素
- **修复**：改为 `modal.addEventListener('click', ...)` 并检查 `e.target === modal`

#### 2. 标签删除后 API Key 列表未刷新 🔄
- **问题**：点击标签删除按钮后，标签被删除但 API Key 列表没有刷新
- **原因**：`onDelete` 回调只调用了 `loadTagList()` 刷新标签列表
- **修复**：添加 `window.loadData()` 调用刷新 API Key 列表

### 技术实现

#### 修改的文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `frontend/src/ui.js` | 修改 | 修复 showConfirmDialog 事件绑定 |
| `frontend/src/tags.js` | 修改 | 标签删除后刷新 API Key 列表 |

## V1.0.8 (2026-03-06) - Toast 参数修复

### 修复内容

#### 修复 Toast 显示类型文字问题 🐛
- **问题**：Toast 提示显示 "success"、"info"、"error" 等类型文字，而不是消息内容
- **原因**：`tags.js` 和 `main.js` 中 `showToast` 函数调用时参数顺序与 `ui.js` 定义不一致
  - `ui.js` 定义：`showToast(message, type)` - 消息在前，类型在后
  - `tags.js` / `main.js` 调用：`showToast('error', '消息')` - 类型在前，消息在后
- **修复**：统一所有调用为 `(message, type)` 顺序

### 技术实现

#### 修改的文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `frontend/src/tags.js` | 修改 | 修复 10 处 showToast 参数顺序 |
| `frontend/src/main.js` | 修改 | 修复 12 处 showToast 参数顺序 |

## V1.0.7 (2026-03-06) - Toast 样式优化

### 优化内容

#### Toast 提示美化 🎨
- **尺寸增大**：最小宽度从 200px 增加到 280px，最大宽度 420px
- **图标放大**：从 16px 增加到 22px，更清晰醒目
- **字体优化**：14px 字体，行高 1.5，提升可读性
- **阴影增强**：多层阴影设计，增强层次感和悬浮感
- **颜色区分**：不同类型 Toast 使用不同背景色和边框
  - success: 绿色系 (rgba(16, 185, 129, 0.15))
  - error: 红色系 (rgba(239, 68, 68, 0.15))
  - info: 蓝色系 (rgba(59, 130, 246, 0.15))
  - warning: 黄色系 (rgba(245, 158, 11, 0.15))
- **动画优化**：使用 cubic-bezier 缓动函数，动画更流畅自然
- **位置调整**：略微提升底部位置，增加 z-index 确保在最上层
- **圆角优化**：使用 var(--radius-lg) 统一圆角风格

## V1.0.5 (2026-03-06) - 桌面端体验打磨

### 新增功能

#### 1. Key 显示/隐藏切换 🔒
- **默认脱敏显示**：API Key 默认以脱敏格式显示（如 `sk-proj-••••••x8`）
- **点击切换明文**：点击眼睛图标可查看完整 Key
- **30 秒自动隐藏**：明文显示 30 秒后自动恢复脱敏状态
- **状态同步**：切换主题/刷新列表时保持当前显示状态

#### 2. 删除确认对话框 ⚠️
- **二次确认**：删除 API Key 时弹出确认对话框，防止误删
- **友好提示**：显示网站名称和"此操作不可恢复"提示
- **键盘支持**：支持 Enter 确认、Esc 取消

#### 3. 空状态优化 🎨
- **首次使用引导**：无密钥时显示友好的引导提示和"添加第一个密钥"按钮
- **无搜索结果提示**：搜索无结果时显示"未找到匹配的密钥"提示
- **图标区分**：根据空状态类型显示不同图标（钥匙/搜索）

#### 4. 键盘快捷键 ⌨️
| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl + N` | 新建密钥 | 打开添加密钥模态框 |
| `Ctrl + F` | 聚焦搜索框 | 快速开始搜索 |
| `Ctrl + E` | 导出数据 | 打开导出对话框 |
| `Ctrl + I` | 导入数据 | 打开导入对话框 |
| `Ctrl + T` | 管理标签 | 打开标签管理对话框 |
| `Ctrl + Enter` | 保存 | 在模态框中保存 |
| `Esc` | 关闭 | 关闭模态框/清除搜索 |
