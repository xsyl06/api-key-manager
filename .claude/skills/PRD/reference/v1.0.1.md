#### 1. 导入功能增强：密钥重新加密与标签ID映射

**需求**: 修复跨设备导入后 API Key 无法解密和标签不显示的问题

**问题描述**:
- 跨电脑导入时，导出的密文使用导出方密钥加密，导入后无法用本地密钥解密
- 跨电脑导入时，标签ID引用不匹配，导致前端无法展示标签

**ASCII 原型图 - 问题分析**:

│   电脑A 导出                          电脑B 导入                         │
│   ┌─────────────────┐               ┌─────────────────┐              │
│   │ master.key (keyA)│               │ master.key (keyB)│              │
│   │ API Key (加密)   │ ── ZIP ──▶    │ API Key (加密)   │              │
│   │ 用 keyA 加密     │               │ 仍用 keyA 加密   │ ❌            │
│   └─────────────────┘               └─────────────────┘              │
│                                            │                            │

**核心代码逻辑**:

```go
// 1. 解密-再加密
for i := range importFile.Items {
    plaintext, _ := crypto.Decrypt(&item.Key, importKey)  // 用导出方密钥解密
    reencrypted, _ := crypto.Encrypt(plaintext, localKey)  // 用本地密钥加密
    item.Key = *reencrypted
}

// 2. 标签ID映射
tagIDMapping := buildTagIDMapping(importFile.Tags, currentData.Tags)
// 结果: { "uuid-A1": "uuid-B1", "uuid-A2": "uuid-A2" }

for i := range importFile.Items {
    remapTagIDs(&importFile.Items[i], tagIDMapping)
}
```

**测试用例**:

| 场景 | 预期结果 |
|------|----------|
| 同一电脑导出→导入 | 成功，API Key 可正常解密 |
| 跨电脑导入 | 成功，API Key 自动用本地密钥重新加密 |
| 导入同名标签 | 复用本地标签ID，不重复创建 |
| 导入新标签 | 创建新标签，保留原ID |
| 冲突解决+标签映射 | 替换后使用本地标签ID |

#### 5. 标签删除逻辑优化

**需求**: 修复删除标签时，选择"仅从标签库删除"选项后，API Key 列表中的标签仍消失的问题

**问题描述**:
- 当前删除标签时有两种选项：
  1. "仅从标签库删除，保留 Key 上的标签标记" → 标签从 `data.Tags` 删除，但 Key 的 `TagIds` 保留
  2. "从标签库和所有 Key 中完全删除该标签" → 标签和关联都删除
- 问题：无论选择哪个选项，API Key 列表中的标签都不显示
- 原因：V1.1 数据模型中，标签显示依赖 `tagMap.get(tagId)` 查找，标签从 `data.Tags` 删除后无法找到

**解决方案**:
- 移除选项，统一按"从标签库和所有 Key 中完全删除"处理
- 简化确认对话框，仅提示标签被多少个 Key 使用
