## 功能规格更新

### 1. 编辑 API Key 弹窗改进 (v1.0.2)

#### 变更前
- 打开编辑弹窗时，API Key 输入框为空
- 需点击"填入当前 Key"按钮才解密并填充原有 Key
- 多一步操作，用户体验不佳

#### 变更后
- 打开编辑弹窗时，自动解密 API Key 并填充到输入框
- 移除"填入当前 Key"按钮
- 新增 API Key 显示/隐藏切换按钮（眼睛图标）
- 保存时将输入框中的值按更新逻辑更新到文件中

#### 技术实现

**前端修改** (`frontend/src/main.js`):

```javascript
async function openEditModal(keyData) {
    // ... 弹窗 HTML 结构
    modal.innerHTML = `
        <div class="form-group">
            <label>API Key *</label>
            <div class="key-input-wrapper" style="position: relative;">
                <input type="password" class="glass-input form-input" name="apiKey"
                       required placeholder="正在加载..." id="editApiKeyInput">
                <button type="button" class="toggle-key-visibility"
                        id="toggleEditKeyVisibility">
                    <i class="ph ph-eye"></i>
                </button>
            </div>
        </div>
    `;

    // 打开弹窗时自动解密并填充 API Key
    const apiKeyInput = modal.querySelector('#editApiKeyInput');
    try {
        currentKeyValue = await DecryptKey(keyData.id);
        apiKeyInput.value = currentKeyValue;
        apiKeyInput.placeholder = 'sk-...';
    } catch (error) {
        apiKeyInput.placeholder = '加载失败，请手动输入';
        showToast('error', '获取 Key 失败：' + error.message);
    }

    // 切换 API Key 显示/隐藏
    toggleBtn.addEventListener('click', () => {
        const isPassword = apiKeyInput.type === 'password';
        apiKeyInput.type = isPassword ? 'text' : 'password';
        toggleBtn.querySelector('i').className = isPassword
            ? 'ph ph-eye-slash' : 'ph ph-eye';
    });

    // 保存时验证 API Key 不能为空
    modal.querySelector('#editKeyForm').addEventListener('submit', async (e) => {
        const apiKey = formData.get('apiKey');
        if (!apiKey || apiKey.trim() === '') {
            showToast('error', 'API Key 不能为空');
            return;
        }
        // ... 调用 UpdateKey 更新
    });
}
```

#### 数据流

```
用户点击编辑按钮
    ↓
打开编辑弹窗 (openEditModal)
    ↓
调用 DecryptKey(keyData.id) 解密 API Key
    ↓
将明文 Key 填充到输入框 (type="password")
    ↓
用户可查看/修改 Key（点击眼睛图标切换显示）
    ↓
点击保存 → 验证 Key 不为空 → 调用 UpdateKey 更新
```

### 2. API Key 列表选中样式优化 (v1.0.2)

#### 问题描述
- **原设计问题**：选中状态使用 `border-left: 3px solid` 直线边框，视觉效果生硬
- **用户体验**：hover 和 selected 状态区分不够明显，缺少动画过渡

#### 设计目标
- 选中或鼠标 hover 时，左侧呈现半圆角形式的高亮样式
- 选中状态有持久阴影，hover 状态临时高亮
- 支持深色和浅色两种主题
- 平滑动画过渡

#### 变更后效果

| 状态 | 视觉效果 | 动画 |
|------|---------|------|
| **默认** | 透明背景，分隔线样式 | - |
| **Hover** | 半透明背景 + 左侧圆弧高亮 + 轻微阴影 | 200ms 渐变 |
| **Selected** | 更深的背景 + 左侧圆弧高亮 + 持久阴影 | 250ms 渐变 |

#### 技术实现

**方案核心**：
1. 使用 `::before` 伪元素创建左侧半圆角高亮条
2. 使用 `border-radius` 和 `box-shadow` 创建圆角和阴影效果
3. hover 和 selected 使用不同透明度/深度区分

**JavaScript 逻辑修改** (`frontend/src/main.js`):

```javascript
// V1.0.1: 选中 Key 项（用于键盘快捷键）- 增加样式切换
function selectKeyItem(keyId) {
    // 清除之前的选中状态
    elements.keyList.querySelectorAll('.key-item').forEach(item => {
        item.classList.remove('selected');
    });

    // 设置新的选中状态
    const item = elements.keyList.querySelector(`[data-id="${keyId}"]`);
    if (item) {
        item.classList.add('selected');
        AppState.selectedKeyId = keyId;
    }
}

// 点击事件绑定 - 点击选中，hover 仅临时高亮
item.addEventListener('click', () => {
    selectKeyItem(keyId);
});

// Hover 状态由 CSS 自动处理，无需 JS 干预
```

#### 设计规格

**颜色规范**：

| 主题 | Hover 背景 | Selected 背景 | 左侧高亮条 | 选中阴影 |
|------|-----------|--------------|-----------|---------|
| 深色 | `rgba(102, 126, 234, 0.08)` | `rgba(102, 126, 234, 0.15)` | `#667eea` (opacity 0.8) | `rgba(102, 126, 234, 0.2)` |
| 浅色 | `rgba(102, 126, 234, 0.06)` | `rgba(102, 126, 234, 0.12)` | `#667eea` (opacity 0.7) | `rgba(102, 126, 234, 0.15)` |

**动画时序**：
```
Hover 进入：0ms 开始 → 200ms 完成 (背景 + 阴影 + 高亮条)
Hover 离开：0ms 开始 → 150ms 完成 (快速恢复)

选中：0ms 开始 → 250ms 完成 (背景 + 阴影 + 高亮条)
取消选中：0ms 开始 → 200ms 完成
```

#### 主题兼容性

**深色主题**（默认）：
- 背景：深色渐变 `#1a1a2e → #16213e`
- 高亮：蓝紫色强调色 `#667eea`
- 阴影：带发光效果

**浅色主题**（`[data-theme="light"]`）：
- 背景：浅灰渐变 `#f5f7fa → #e4e8ec`
- 高亮：同色相，降低透明度
- 阴影：更柔和，不发光
